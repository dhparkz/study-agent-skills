# 4장 배포 및 공유

스킬은 MCP와의 통합 과정을 더 **완성품**에 가깝게 만듭니다. 사용자가 커넥터를 비교할 때, 스킬이 함께 제공되는 쪽이 더 빠르게 가치에 도달하므로 MCP-only 대비 경쟁력이 생깁니다.

## 현재 배포 모델(2026년 1월)

### 개별 사용자가 스킬을 얻는 방법:
1. 스킬 폴더 다운로드
2. (필요하면) 폴더를 zip으로 압축
3. Claude.ai에 업로드: Settings > Capabilities > Skills
4. 또는 Claude Code의 skills 디렉터리에 배치

### 조직(워크스페이스) 단위 스킬:
- 관리자는 워크스페이스 전체에 스킬을 배포할 수 있습니다(2025-12-18 출시).
- 자동 업데이트
- 중앙 관리

## 오픈 표준(An open standard)

[Agent Skills](https://agentskills.io/home)를 오픈 표준으로 공개했습니다. MCP와 마찬가지로, 우리는 스킬이 다양한 도구와 플랫폼에서 활용 가능해야 한다고 생각합니다. 즉, Claude를 쓰든 다른 AI 플랫폼을 쓰든 같은 스킬이 동작해야 합니다. 다만 일부 스킬은 특정 플랫폼의 기능을 최대한 활용하도록 설계되며, 이 경우 authors는 `compatibility` 필드에 명시할 수 있습니다. 우리는 생태계 구성원들과 표준을 함께 발전시키고 있고, 초기 채택 사례에 고무되어 있습니다.

## API로 스킬 사용하기(Using skills via API)

애플리케이션, 에이전트, 자동화 워크플로 등 프로그램 방식으로 스킬을 활용하는 유스케이스에서는 API가 스킬 관리와 실행을 직접 제어할 수 있게 해줍니다.

핵심 기능:
- 스킬 목록/관리를 위한 `/v1/skills` 엔드포인트
- Messages API 요청에 `container.skills` 파라미터로 스킬 추가
- Claude Console을 통한 버전 관리/관리
- 커스텀 에이전트를 만들기 위한 Claude Agent SDK와 함께 동작

### API로 스킬을 사용할지, Claude.ai에서 사용할지 기준:

| 사용 사례(Use Case) | 권장 환경(Best Surface) |
|---|---|
| 최종 사용자가 스킬을 직접 사용 | Claude.ai / Claude Code |
| 개발 중 수동 테스트/반복 개선 | Claude.ai / Claude Code |
| 개인의 ad-hoc 워크플로 | Claude.ai / Claude Code |
| 애플리케이션에서 스킬을 프로그램 방식으로 사용 | API |
| 대규모 프로덕션 배포 | API |
| 자동화 파이프라인/에이전트 시스템 | API |


**참고:** API에서 스킬을 쓰려면 Code Execution Tool(beta)이 필요합니다. 스킬 실행에 필요한 보안 실행 환경을 제공합니다.

구현 상세는 아래를 참고하세요:
- [Skills API 빠른 시작(Quickstart)](https://docs.claude.com/en/docs/agents-and-tools/agent-skills/quickstart)
- [커스텀 스킬 만들기(Create custom skills)](https://docs.claude.com/en/api/skills/create-skill)
- [Agent SDK에서 스킬 사용(Skills in the Agent SDK)](https://docs.claude.com/en/docs/agent-sdk/skills)

### 현재 권장하는 접근 방식:

먼저 GitHub에 공개 레포로 스킬을 호스팅하고, 설치 방법이 포함된 명확한 README(사람을 위한 문서 — 스킬 폴더 안에는 README.md를 넣지 않는다는 점에 유의)와 스크린샷 포함 사용 예시를 제공하세요. 그다음 MCP 문서에 스킬 링크 및 "함께 쓰는 가치" 설명, 빠른 시작 가이드를 추가하세요.

#### 1. GitHub에 호스팅
  – 오픈소스 스킬을 위한 퍼블릭 레포
  – 설치 방법이 포함된 명확한 README
  – 사용 예시와 스크린샷

#### 2. MCP 레포 문서에 정리
  – MCP 문서에서 스킬로 링크
  – 둘을 함께 쓸 때의 가치 설명
  – 빠른 시작 가이드 제공

#### 3. 설치 가이드 만들기
```
## [Your Service] 스킬 설치 방법

1. 스킬 다운로드:
   - 레포 Clone: `git clone https://github.com/yourcompany/skills`
   - 또는 Releases에서 ZIP 다운로드

2. Claude에 설치:
   - Claude.ai > Settings > Skills 열기
   - "Upload skill" 클릭
   - (zip된) 스킬 폴더 선택

3. 스킬 활성화:
   - [Your Service] 스킬을 켜기(Enable/Toggle on)
   - MCP 서버가 연결되어 있는지 확인

4. 테스트:
   - Claude에게 요청: "[Your Service]에 새 프로젝트 세팅해줘"
```

## 스킬 포지셔닝(Positioning your skill)

스킬을 어떻게 설명하느냐에 따라, 사용자가 가치를 이해하고 실제로 시도할지 결정됩니다. README, 문서, 마케팅에 스킬을 소개할 때 아래 원칙을 기억하세요.

**기능이 아니라 결과에 집중**

✅ 좋은 예:
```yaml
"ProjectHub 스킬은 팀이 페이지/데이터베이스/템플릿까지 포함한 프로젝트 워크스페이스를 몇 초 만에 세팅하도록 도와줍니다. 수동 설정에 30분 쓰던 일을 줄일 수 있습니다."
```

❌ 나쁜 예:
```yaml
"ProjectHub 스킬은 YAML 프론트매터와 Markdown 지침으로 구성된 폴더이며, MCP 서버 도구를 호출합니다."
```

**MCP + skills 스토리를 강조**
```yaml
"MCP 서버는 Claude가 Linear 프로젝트에 접근할 수 있게 해주고, 스킬은 팀의 스프린트 계획 워크플로를 가르칩니다. 함께 쓰면 AI 기반 프로젝트 관리가 가능합니다."
```
