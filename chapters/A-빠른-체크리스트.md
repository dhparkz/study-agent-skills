# 참고 A: 빠른 체크리스트(Quick Checklist)

이 체크리스트는 스킬을 업로드 전/후에 검증하는 데 사용합니다. 더 빠르게 시작하려면 skill-creator로 초안을 만든 뒤 이 리스트로 빠진 부분을 점검하세요.

## 시작 전
- 구체적인 유스케이스 2~3개를 정했는가
- 필요한 도구를 파악했는가 (내장 도구 또는 MCP)
- 이 가이드와 예시 스킬을 검토했는가
- 폴더 구조를 계획했는가

## 개발 중
- 폴더가 kebab-case로 명명되었는가
- SKILL.md 파일이 존재하는가 (정확한 철자)
- YAML 프론트매터에 `---` 구분자가 있는가
- name 필드: kebab-case, 공백/대문자 없음
- description에 무엇(WHAT)과 언제(WHEN)가 포함되었는가
- XML 태그(`< >`)가 어디에도 없는가
- 지침이 명확하고 실행 가능한가
- 에러 핸들링이 포함되었는가
- 예시가 제공되었는가
- references가 명확히 링크되었는가

## 업로드 전
- 명확한 작업 요청에서 트리거 테스트 완료
- 바꿔 말한 요청에서 트리거 테스트 완료
- 관련 없는 주제에서 트리거되지 않는 것을 확인
- 기능 테스트 통과
- 도구 통합이 동작하는지 확인 (해당 시)
- .zip 파일로 압축 완료

## 업로드 후
- 실제 대화에서 테스트
- 소 트리거/과 트리거 모니터링
- 사용자 피드백 수집
- description과 지침을 반복 개선
- metadata의 version 업데이트
