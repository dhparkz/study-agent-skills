# 1장 기초

## 스킬이란?

스킬은 다음을 포함하는 **폴더**입니다.

- **SKILL.md (필수)**: YAML 프론트매터가 포함된 Markdown 지침
- **scripts/ (선택)**: 실행 가능한 코드(Python, Bash 등)
- **references/ (선택)**: 필요할 때 로드되는 문서
- **assets/ (선택)**: 출력에 사용되는 템플릿, 폰트, 아이콘

## 핵심 설계 원칙

### 점진적 공개(Progressive Disclosure)

스킬은 3단계 시스템을 사용합니다.

- **1단계(YAML 프론트매터)**: Claude의 시스템 프롬프트에 **항상 로드**됩니다.
  전체 내용을 컨텍스트에 다 넣지 않고도, Claude가 **언제** 스킬을 써야 하는지 알 수 있을 만큼의 정보만 제공합니다.
- **2단계(SKILL.md 본문)**: Claude가 현재 작업과 관련 있다고 판단할 때 로드됩니다.
  전체 지침과 가이드를 담습니다.
- **3단계(링크된 파일)**: 스킬 디렉터리 안에 함께 묶인 추가 파일들로, Claude가 필요할 때만 찾아가며 확인할 수 있습니다.

이 점진적 공개 방식은 토큰 사용량을 최소화하면서도 전문성을 유지합니다.

### 조합성(Composability)

Claude는 여러 스킬을 동시에 로드할 수 있습니다. 내 스킬은 다른 스킬과 **함께 잘 동작**해야 하며, "내 스킬만 존재한다"는 가정을 하면 안 됩니다.

### 이식성(Portability)

스킬은 Claude.ai, Claude Code, API 전반에서 동일하게 동작합니다. 스킬을 한 번 만들면, 스킬이 필요로 하는 의존성이 지원된는 환경에서 수정 없이 모든 곳 에서 사용할 수 있습니다.

## MCP 빌더를 위한: Skills + Connectors

> 💡 MCP 없이 단독 스킬을 만드는 중이라면 [기획 및 설계](02-기획-및-설계.md)로 건너뛰어도 됩니다. 필요하면 나중에 다시 돌아올 수 있습니다.

이미 [동작하는 MCP 서버](https://support.claude.com/ko/articles/10949351-getting-started-with-local-mcp-servers-on-claude-desktop
)가 있다면, 가장 어려운 부분은 이미 끝났습니다. 스킬은 그 위에 올라가는 **지식 레이어**로, 여러분이 이미 알고 있는 워크플로와 모범 사례를 담아 Claude가 매번 일관되게 적용하도록 해줍니다.

### 주방 비유(The kitchen analogy)

- MCP는 **전문 주방**을 제공합니다: 도구, 재료, 장비에 접근할 수 있게 해줍니다.
- 스킬은 **레시피**를 제공합니다: 가치 있는 결과물을 만들기 위한 단계별 방법을 제공합니다.


둘을 함께 쓰면, 사용자가 모든 단계를 일일이 고민하지 않아도 복잡한 작업을 수행할 수 있습니다.

### 함께 동작하는 방식

| MCP(연결성) | Skills(지식) |
|---|---|
| Claude를 서비스(Notion, Asana, Linear 등)에 연결 | Claude가 서비스를 효과적으로 쓰는 방법을 가르침 |
| 실시간 데이터 접근 및 도구 호출 제공 | 워크플로와 모범 사례를 담아둠 |
| Claude가 할 수 있는 것 | Claude가 어떻게 해야 하는지 |

### MCP 사용자에게 왜 중요한가

**스킬이 없으면:**
- 사용자는 MCP를 연결해도 다음에 뭘 해야 할지 모름
- "이 통합으로 X를 어떻게 해요?" 같은 지원 티켓이 늘어남
- 매 대화가 항상 처음부터 시작됨
- 매번 프롬프트가 달라 결과가 들쭉날쭉함
- 실제 문제는 워크플로 가이드 부족인데, 사용자는 커넥터 탓을 하게 됨

**스킬이 있으면:**
- 필요할 때 미리 만들어둔 워크플로가 자동으로 활성화됨
- 일관되고 신뢰 가능한 도구 사용
- 모범 사례가 모든 상호작용에 내장됨
- 통합의 학습 곡선이 낮아짐
